## 生存模式不能吃 + 配方无效 修复

### 生存模式不能吃
- **原因一**：物品通过 `registerSimpleItem` 注册为普通 `Item`，而非 `SuperNutritionFoodItem`，自定义吃食物逻辑从未生效
- **原因二**：没有 `.food()` 属性，`isEdible()` 返回 false → 原版在生存模式不识别为食物 → 右击无反应
- **修复**：
  1. `ITEMS.register("super_nutrition_food", () -> new SuperNutritionFoodItem(...))` — 注册自定义 Item 类
  2. 添加 `.food(...).alwaysEdible()` — 标记为永远可食用
  3. `use()` 中去掉 `canEat(false)` 检查 — 直接 `startUsingItem(hand)`
  4. 添加 `getUseAnimation()` 返回 `UseAnim.EAT`

### 配方无效
- **原因**：recipe JSON 中多个物品 ID 在 TFC 4.2.4 中不存在或名称不同
- **修复对照**：
  | 旧 ID | 新 ID | 说明 |
  |------|------|------|
  | `tfc:clay_ball` | `tfc:clay/entisol` | TFC没有clay_ball，粘土是方块形式 |
  | `tfc:ore/guano` | `tfc:groundcover/guano` | 蝙蝠粪是地面覆盖物 |
  | `tfc:pebbles` tag | `c:stones/loose` | 松散岩石的通用标签 |
  | `item: x` | `id: x` | 新版 Minecraft 1.21 recipe 格式 |
- **验证方法**：从 TFC jar 中提取 `zh_cn.json` 语言文件和 `tags/item/` 目录，逐一核对物品 ID

## "只能吃一次" + "EMI 看不到配方" 修复

### 只能吃一次
- **根因**：`applyTFCNutrition()` 只调用了 `nutrition().addNutrients()` + `addThirst()`，
  但**从未更新原版饥饿值/饱和度**。TFC 的 `PlayerMixin` 完全替换了 `Player.eat()`，
  非 IFood 物品的 FoodProperties 永远不会被应用。
- **修复**：
  1. 改用 `playerInfo.eat(FoodData)` 一站式处理所有食物效果
  2. `PlayerInfo.eat(FoodData)` 内部会依次调用：
     - `addThirst(water)` → 口渴值
     - `food.eat(hunger, sat)` → 原版饥饿值/饱和度（绕过 TFC 的拦截）
     - `nutrition.addNutrients(foodData, getFoodLevel())` → TFC 五种营养
     - `nutrition.setHungerAndUpdate()` → 同步饥饿跟踪
  3. 只在服务端调用 `applyFoodEffects()`（`if (!level.isClientSide)`）
- **关键发现**：TFC 的 `IPlayerInfo.eat(ItemStack)` 默认方法只处理 IFood 物品，
  非 IFood 物品直接返回空。`PlayerInfo.eat(FoodProperties)` 和 `PlayerInfo.eat(int,float)` 都是空方法。

### EMI 看不到配方
- **根因**：`"category": "food"` 在 Minecraft 1.21.1 的 `CraftingBookCategory` 枚举中不存在！
  有效值只有 `building`/`redstone`/`equipment`/`misc`。无效 category 导致配方被静默忽略。
- **修复**：改为 `"category": "misc"`
- **确认**：所有配方材料 ID（`tfc:straw`, `tfc:clay/entisol`, `tfc:groundcover/guano`,
  `tfc:dirt/entisol`, `c:stones/loose`, `minecraft:stick`）均在 TFC 4.2.4 中有效存在

## "口渴值或五种营养值满时无法食用" 修复

- **问题**：当 TFC 口渴值达到 100/100 或五种营养值均达到 1.0（100%）后，超级营养食物无法再食用
- **排查过程**：
  1. 检查 `PlayerInfo.eat(FoodData)` 字节码 — 无 guard clause，理论上不会拒食
  2. 检查 `ForgeEventHandler.onPlayerRightClickItem` — 仅委托 `InteractionManager.onItemUse`，不拦截食物
  3. 检查 `InteractionManager.onItemUse` — 处理 UseOnContext（对物品使用），非空手右键吃食物
  4. 检查所有 TFC mixin（PlayerMixin/MobMixin/EntityMixin/ItemStackMixin）— 均不拦截 `startUsingItem` 或 `finishUsingItem`
  5. 结论：`playerInfo.eat(FoodData)` 内部的 `addThirst()` 或联动逻辑可能在满值时跳过；无法确认具体拦截点，但确认绕过方案可行
- **修复**：放弃 `playerInfo.eat(FoodData)` 一站式调用，改为直接操作底层 API：
  1. `playerInfo.addThirst(thirst)` — 直接添加口渴值
  2. `player.getFoodData().eat(hunger, sat)` — 直接操作原版 FoodData（绕过 TFC PlayerMixin 替换的 Player.eat()）
  3. `playerInfo.nutrition().addNutrients(foodData, foodLevel)` — 直接添加 TFC 营养
  4. `playerInfo.nutrition().setHungerAndUpdate(foodLevel)` — 同步饥饿跟踪
  5. 使用 `player.getFoodData().getFoodLevel()` 替代 `IPlayerInfo.getFoodLevel()`（后者是 PlayerInfo 实现类方法，不在接口中）
- **注意**：`Item.isEdible()` 在 Minecraft 1.21.1 不是可覆盖方法，`alwaysEdible` 由 `FoodProperties.canAlwaysEat()` 控制（通过 `.food().alwaysEdible()` 已设置）
